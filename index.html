Project Mission

URLGuard AI is designed to ingest IPDR/PCAP-derived HTTP traffic, detect attack attempts, and separate successful exploit chains from failed probes. It helps SOC teams visualize, query, and export incident intelligence in CSV/JSON.

Required Modules

Data Ingestion

Import simulated datasets, external PCAP, and parsed URL logs from multiple sources.

Attack Detection Engine

Detect known and emerging URL attack patterns using rules + ML-assisted classifiers.

Success vs Attempt Classifier

Tag events as attempt/success with confidence scoring and forensic evidence links.

Threat Visualization

Interactive charts for attack type, source IP range, timeline, and severity distribution.

Query & Search Console

Filter by attack family, endpoint, IP/CIDR, time range, and impact indicators.

Export & Reporting

Download findings in CSV/JSON and generate shareable incident reports.

Workflow

1. Generate simulated attacks using SQLMap, Burp Suite, XSStrike, Commix.
2. Ingest packet and log data, normalize URL/request attributes.
3. Detect attack signatures and anomalous URL behavior.
4. Correlate indicators to identify confirmed successful attacks.
5. Investigate and export actionable intelligence for response teams.
